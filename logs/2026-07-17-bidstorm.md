# Build Log: Bidstorm
**Date:** 2026-07-17
**Status:** deployed

## Idea Source
**Invented.** IDEAS.md was empty. `card` was the only empty category in
registry.json, and a bidding/auction game is close to ideal for this factory: the
netcode is one small integer per round, it is turn-based (so latency is
irrelevant), and it is deterministic from a seed. The core is the public-domain
auction-game family (Goofspiel/GOPS, Merrill Flood, 1950s) — a well-made classic,
which the brief explicitly allows — rebuilt with an original identity, three
original modes, and a results screen that is the real design work.

## Game Details
- **Name / repo:** Bidstorm / ben-gy/bidstorm
- **Genre:** card
- **Core loop:** A prize flips; everyone secretly commits one card from an
  *identical* 1–13 hand; highest bid takes the pot; every card played is burned
  forever. A tie for the lead wins nothing and carries the pot. Every hand is
  public, so it is a decision rather than a guess.
- **Multiplayer:** live P2P, host-authoritative star, 2–6 players. Plus
  async-seed (share a link carrying seed + mode; a friend plays the same deal).
- **Stack / render:** vanilla-ts / dom (+ a canvas overlay for particles/shake)
- **Engine modules used:** net, rematch, lobby, rng, sound, storage, mobile,
  identity. Deliberately NOT loop.ts (turn-based, no fixed-timestep sim) or
  input.ts (tap-only; no D-pad).

## Design decisions worth recording
- **Solo runs with no round clock.** The 30s clock exists so a table is never held
  hostage by a player who left. Solo has nobody to wait for, so a clock there
  would only take the round away from someone who was thinking — a punishment
  with no purpose in a game that is entirely thinking. Found by playing it: the
  clock expired mid-screenshot and auto-played my lowest card.
- **Three modes, on different axes.** Classic (information: none), Foresight
  (lookahead 2), Blitz (deck 7, ties VOID instead of carrying, 12s). The
  tie rule is the interesting one: in Classic you tie by accident and get rich;
  in Blitz you tie *on purpose* to burn a prize your rival was about to take.
- **The mode is netcode, not a setting.** It decides the DECK SIZE, so a mode each
  peer read from its own menu is two peers dealing different games off the same
  seed and never finding out. `rematch.ts` gained `roundOpts`: the host's settings
  travel frozen inside the round start, exactly like the roster. The share link
  carries the mode for the same reason.
- **Live bids are not in the bytes.** While a round is open the host sends each
  peer a snapshot masked to *that peer* (`toSnapshot(g, forPlayer)`). Sending all
  the cards and hiding them in the UI would put every opponent's bid one devtools
  console away, and nobody would ever find out. Disclosed honestly in About: a
  modified host could still peek.
- **The results screen shows everyone, and what everyone missed.** Because bids
  are simultaneous, the rest of the table is fixed regardless of what one player
  did — so "the cheapest card in your hand that beats everyone else's bid" is a
  genuinely correct answer in hindsight, not a hand-wave. That yields two honest
  columns: overspend and missed steals.

## Juice
Prize card slams in with an eased overshoot and a shake scaled to its value; cards
lift and tilt when armed; commits land face-down as coloured backs so the wait has
visible progress; the reveal flips bids 90ms apart, pulses the winner, and fires a
particle burst in their colour; a tie cracks red and shatters both cards; the last
round gets a longer beat. 3·2·1·GO before each round. Procedural SFX throughout;
`prefers-reduced-motion` drops shake/particles and flattens the animations whose
start state would otherwise leave an element invisible.

## Errors & Resolutions
- **Two tests failed on false premises, and both were interesting.** (a) "Player 0
  wins every round" is *arithmetically impossible*: both players hold all thirteen
  cards, so whoever plays the 13 cannot be beaten that round — and both hold a 13.
  Replaced with the real invariant: points are conserved (scores + carry == 91) on
  any line of play, on any seed. (b) An analysis test assumed a player could beat a
  12 while holding only 1–12. Both were my tests being wrong, not the game.
- **`source-hygiene.test.ts` caught a literal ESC byte in its own file** while I was
  writing the test that forbids literal control bytes. Rewrote the file. The best
  possible argument for that test existing.
- **`tsc` rejected `phase === 'bidding'` after `resolve()`** — it had narrowed the
  type to `'reveal'` and cannot see the mutation. Rewrote the branch to test
  `round < ROUNDS` instead of casting the check away.
- **Stale armed card carried between rounds** (found by playing, not by tests): a
  card armed for the last prize stayed lifted into the next one. Now cleared on
  every round change.
- **Mid-build the tree gained a modes + countdown feature** (`src/modes.ts`,
  `src/countdown.ts`) that was only half-integrated and left the build broken.
  Completed the integration rather than reverting: mode threaded through
  Match/UI/results/share, `roundOpts` added to rematch so it travels frozen,
  countdown wired with `Match.beginPlay()` so the 3·2·1 doesn't eat the round
  clock, plus 30 new tests (`modes.test.ts`, `countdown.test.ts`, roundOpts cases).
- Nostr relay warnings (`relay.nostromo.social` blocked, `relay.nostraddress.com`
  auth-required) are noisy in console but harmless — the mesh formed on the other
  relays and every gate passed live.
