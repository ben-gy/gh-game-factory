## What was wrong

Three field failures, all traced to real code before anything was written:

1. **Peers never appearing in rooms.** `createNet` passed no `turnConfig`, so ICE
   was STUN-only. A phone on carrier CGNAT or a locked-down network sees the peer
   in signaling and never opens a data channel. Mobile-first games ⇒ likely the
   dominant cause.
2. **A joiner stealing host.** A 2.5s settle timeout let a joiner self-elect on a
   roster of one, and the min-id tie-break then handed it a live room — about
   half the time, mid-game, holding none of the state.
3. **Players "ejected" at round start.** The host froze the roster from its
   partial view of a still-forming mesh, and Trystero only delivers to channels
   that are *already* open, so a peer whose channel opened late never learned the
   round began.

## Phase 1 — engine v1.1.0

Epoch host election, start-protocol hardening, TURN injection, diagnostics.
94 tests, typecheck clean. Full detail in the engine's `CHANGELOG.md`.

**An adversarial review (6 lenses, every finding put to a refutation pass) caught
five real defects before any of it shipped** — three room-breaking, and two
present in the handoff's own reference implementation:

| Defect | Why it mattered |
|---|---|
| Promotion nobody else made | Trystero drops a peer on a *transient* `disconnected`, per connection, so one peer can be the only one that saw the host leave. It promoted a survivor that never learned it won, then dropped the real host's announces as stale — stranded forever, no round starts. |
| Min-id beat a real claim | Rosters differ during a transfer, so a locally-elected host is a belief, not a fact. A survivor could pin itself to a peer nobody was hosting from, and re-pin on every announce. |
| Promoted host deadlock | The `cur` catch-up only trusted `from === net.host()`; a promoted host's only trusted peer is itself, so it sat at round 0 while everyone voted for round N+1 — no peer could ever start another round. |
| Spectator ready button inert | `vote()` bailed on `phase === 'playing'`, true for an unseated peer too. A player excluded from one round was excluded from every round — the original failure made *permanent* by its own fix. |
| TURN was one-sided | Trystero builds one global connection pool from the *first* `joinRoom`'s config. A turnless `__presence`/`__board` mesh created first left the game room's initiating half STUN-only. Fixed with `setTurnConfig()`, called once at boot. |

Every fix has a regression test, each **verified to fail when the fix is
reverted**.

### Files deliberately kept game-side

Not failures — genuine game content the engine cannot yet express. Each is
recorded in `EXPANSION_IDEAS.md` under "Engine":

- **`sound.ts`** (8 games) — game-specific patch tables and a `pitch` argument.
  `createSfx()` takes no config and `SfxName` is a closed union of platformer
  cues, so importing it would play "coin" when a card lands.
- **`lobby.ts`** (6 games) — a strict superset carrying the public-rooms browser,
  IP-exposure disclosures, and a host mode-picker slot. View-only: their netcode
  comes from the package, so **all v1.1.0 fixes apply**. snake-royale and
  gravity-golf also had the new takeover offer and `?netdebug=1` overlay ported in.
- **`drag.ts`** (deepwatch) — game-tuned thresholds.
- **`feedback.ts`** (everywhere) — not engine code; a generated file the hub's
  backfill distributes. Left alone deliberately.
